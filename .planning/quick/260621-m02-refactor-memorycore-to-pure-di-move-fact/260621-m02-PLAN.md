# Quick Task 260621-m02: MemoryCore pure DI

## Problem

`MemoryCore.__init__(backend: BackendPort)` is already pure dependency injection — every
operation method composes only the five `BackendPort` primitives via `self._backend`, so it
already works with ANY conforming backend. But three *convenience factory classmethods*
(`in_memory`, `open`, `from_connection`) name the two first-party backends inside `core.py`.
Goal (user decision: option 2): `core.py` must have **zero** backend/ladybug references; the
convenience constructors move out to free functions and DI is the only construction path in core.

## Scope

Mechanical relocation. **No behavior change.** The three constructors keep identical bodies and
signatures, just as module-level free functions in a new `doxastica/factories.py`, re-exported
from the package root.

## Tasks

### Task 1 — Create `src/doxastica/factories.py`

- **files:** `src/doxastica/factories.py` (new)
- **action:** Create the module with three free functions, bodies copied verbatim from the
  current `MemoryCore` classmethods (minus the `cls(...)` → `MemoryCore(...)` change):
  - `in_memory() -> MemoryCore` — `from doxastica.backends.memory import InMemoryBackend`
    (top-level import is allowed; memory backend is zero-dep), returns `MemoryCore(InMemoryBackend())`.
  - `open(path: str, *, namespace: str = "dx") -> MemoryCore` — **function-local**
    `from doxastica.backends import ladybug` (D-02), returns
    `MemoryCore(cast("BackendPort", ladybug.LadybugBackend.open(path, namespace=namespace)))`.
  - `from_connection(conn: object, *, namespace: str = "dx") -> MemoryCore` — **function-local**
    ladybug import, returns the `owns_conn=False` LadybugBackend wrapped in `MemoryCore`
    (keep the existing `# pyright: ignore[reportArgumentType]`).
  - Module docstring: explain this is the convenience-constructor layer (SQLAlchemy
    `create_engine` analog, D-01); the ONLY place outside the backends package that names a
    backend; ladybug imports are function-local (D-02) so `import doxastica` never chain-loads
    the driver. Import `MemoryCore` from `doxastica.core` and `BackendPort` under `TYPE_CHECKING`.
  - Keep `from __future__ import annotations`.
- **verify:** `python -c "import doxastica.factories"` succeeds; `factories.py` has no
  module-level `ladybug` import.
- **done:** New module exists with the three free functions.

### Task 2 — Strip the classmethods from `src/doxastica/core.py`

- **files:** `src/doxastica/core.py`
- **action:** Delete the `in_memory`, `open`, and `from_connection` classmethods (lines ~160-201).
  Keep `__init__`, `unit_of_work`, and everything else unchanged. Scrub backend/ladybug mentions
  from the **module docstring** and the **class docstring** so `core.py` no longer references
  `InMemoryBackend`, `LadybugBackend`, `ladybug`, `in_memory()`, `open()`, or `from_connection()`.
  Point the construction note at `doxastica.factories` instead. Remove the now-irrelevant D-02
  `TYPE_CHECKING` comment about the factories (lines ~67-68). The `BackendPort` TYPE_CHECKING
  import stays (still used in the `__init__` annotation).
- **verify:** `grep -n ladybug src/doxastica/core.py` returns nothing; basedpyright clean.
- **done:** `core.py` has zero backend-specific references.

### Task 3 — Re-export from `src/doxastica/__init__.py`

- **files:** `src/doxastica/__init__.py`
- **action:** Add `from doxastica.factories import from_connection, in_memory, open` and add
  `"from_connection"`, `"in_memory"`, `"open"` to `__all__` (keep it sorted as the file is).
- **verify:** `python -c "import doxastica; doxastica.in_memory()"` returns a working core.
- **done:** The three factories are importable from the package root.
