# Quick Task 260621-m02: MemoryCore pure DI Summary

Moved the three convenience factory classmethods (`in_memory`, `open`, `from_connection`) off
`MemoryCore` into a new `doxastica.factories` module as module-level free functions (verbatim
bodies), leaving `MemoryCore(backend: BackendPort)` as the sole, backend-blind constructor and
re-exporting the three factories from the package root. No behavior change.

## What changed

- **New `src/doxastica/factories.py`** â€” the SQLAlchemy `create_engine` analog (D-01). Three free
  functions with bodies copied verbatim from the old classmethods (only `cls(...)` â†’ `MemoryCore(...)`):
  - `in_memory() -> MemoryCore` â€” module-local `InMemoryBackend` import (zero-dep, allowed).
  - `open(path, *, namespace="dx") -> MemoryCore` â€” **function-local** `ladybug` import (D-02),
    keeps the `cast("BackendPort", ...)` wrap.
  - `from_connection(conn, *, namespace="dx") -> MemoryCore` â€” function-local `ladybug` import,
    `owns_conn=False`, keeps the `# pyright: ignore[reportArgumentType]` on the `LadybugBackend`
    construction.
  - `from __future__ import annotations` at top; `BackendPort` imported under `TYPE_CHECKING`;
    heavy module + function docstrings explaining the `create_engine` analog (D-01) and the
    function-local-ladybug rationale (D-02). `factories.py` joins the backend-blind spine and
    passes the import-purity module-level scan.
- **`core.py`** â€” deleted the three classmethods. Scrubbed ALL `ladybug` / backend references from
  the module docstring, the class docstring, and the remaining method docstrings/comments
  (rephrased generically to "the reference backend" / "a STRING-column backend" / "the optional
  driver") so `grep -n ladybug src/doxastica/core.py` returns nothing. Removed the now-unused
  `cast` import and the obsolete D-02 `TYPE_CHECKING` factory comment. The `BackendPort`
  `TYPE_CHECKING` import stays (used in the `__init__` annotation). Class docstring now points at
  `doxastica.factories` for construction.
- **`__init__.py`** â€” `from doxastica.factories import from_connection, in_memory, open`; added
  `"from_connection"`, `"in_memory"`, `"open"` to `__all__` (ASCII-sorted, lowercase after the
  uppercase-starting names, matching the file's existing sort).
- **`backends/ladybug.py`** â€” fixed two stale docstring cross-references
  (`MemoryCore.from_connection` / `MemoryCore.open`) to point at `doxastica.factories.*` so the
  whole-task `grep -rn 'MemoryCore\.(in_memory|open|from_connection)' src tests` is clean.
- **Tests (Task 4)** â€” `test_import_purity.py`: added `"factories"` to the module-level-scan
