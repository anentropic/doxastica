Create docs/src/tutorials/cluedo-detective.md as a complete intermediate (~20 min) Diataxis
tutorial implementing the 12-step arc from the approved plan (zany-breeze), section "Tutorial
arc". Match the established tutorial voice, front-matter shape (title/description/difficulty:
intermediate), admonition style (`!!! info`, `!!! tip`, `!!! warning`), and one mermaid
flowchart — mine agent-team-memory.md for these conventions and first-belief-store.md for tone.

Frame the two belief kinds up front: CERTAIN facts (your own hand, a card you were shown) live
in the world scope and can never be wrong; the WORKING THEORY of the crime (culprit/weapon/room)
lives in a belief scope `"theory"` and is provisional and repeatedly overturned. This certain-vs-
provisional split IS the Story B spine — make it explicit in the opening.

Cluedo → primitive mapping (put this as a table, per the approved plan):
  - your own hand + a witnessed card -> world-scope belief (WORLD_SCOPE_ID) — certain, never retracted
  - current theory (culprit/weapon/room) -> belief in scope `"theory"` — provisional, superseded
  - a defeasible inference (e.g. "assume Plum has the Rope") -> belief in `"theory"` + DERIVED_FROM edge to its basis
  - a conclusion drawn from that inference -> belief in `"theory"`, DERIVED_FROM the inference — goes stale when the basis is retracted
  - "what I believed at accusation time" -> get_scope_at(theory, as_of=accusation_event)
  - (optional, brief) modeling an opponent's knowledge -> a second belief scope `"opp:Scarlett"`

Steps to write (each ties to one real operation; narrate, then a runnable code block, then the
printed/expected output as in the existing tutorials):
  1. Build the core: `core = MemoryCore(InMemoryBackend())`; import surface below.
  2. Seed the world scope with what you KNOW: `core.revise(WORLD_SCOPE_ID, "my-hand", {...}, source_event_id=uuid7())` and each witnessed card (e.g. belief_id `"seen:mustard"`). Frame: ground truth — certain, never revised away.
  3. Form a first theory in scope `"theory"`: revise `culprit` (e.g. "Mustard"), `weapon`, `room` to your best current guess. Frame: this is a belief, not a fact.
  4. A defeasible inference WITH provenance: `assume = core.revise("theory", "assume-plum-has-rope", ..., source_event_id=uuid7())` capturing its BeliefState; then derive `weapon_state = core.revise("theory", "weapon", "Wrench", ...)` and `core.add_edge(weapon_state.state_id, assume.state_id, EdgeType.DERIVED_FROM)`. Edge direction is dependent -> basis: the weapon conclusion now RESTS ON the assumption.
  5. Contradiction #1 — the theory is overturned: you are shown the Mustard card. YOU recognize it refutes culprit="Mustard" and call `core.revise("theory", "culprit", "Plum", source_event_id=uuid7())`. Then `core.get_revision_chain("culprit")` prints the crossed-out notebook: Mustard -> Plum, prior superseded, nothing deleted. (Note in prose: get_revision_chain is by belief_id; keep `culprit` unique across scopes in this tutorial.)
  6. Contradiction #2 — a false inference and its fallout: new evidence disproves the assumption. `core.contract("theory", "assume-plum-has-rope", source_event_id=uuid7())`, then `core.get_impact(assume.state_id)` returns the now-stale weapon="Wrench" belief that was DERIVED_FROM it — the cue to re-derive. Show `query_scope(..., include_retracted=True)` to reveal the retracted tail.
  7. You can't un-see a card: `try: core.contract(WORLD_SCOPE_ID, "seen:mustard", source_event_id=uuid7())` -> catch `WorldScopeContractionError`. This is WHY world scope is privileged: certainty is not revisable.
  8. The audit trail: `core.get_scope_at("theory", accusation_event)` rewinds `culprit` back to "Mustard" — reconstruct exactly what you believed at accusation time (capture `accusation_event = uuid7()` at the moment of the first theory in step 3 and reuse it).
  9. (Optional, brief) Theory of mind = another scope: a second belief scope `"opp:Scarlett"` records what you infer Scarlett has seen. Frame as the multi-scope extension earning its place beyond single-agent Kumiho — keep it short and subordinate to the revision story.
  10. "What doxastica does not do" (REQUIRED honesty section): doxastica does NOT deduce, solve constraints, or check that values are mutually consistent. YOU notice the shown card contradicts the theory and call revise; YOU decide an inference is dead and call contract. doxastica is the append-only, revisable, traceable MEMORY that supersedes cleanly, cascades impact, and reconstructs history — not the reasoner.
  11. Verification: one runnable closing script mirroring the existing tutorials' closing script, with assertions — chain length after supersede, get_impact returns the derived belief, WorldScopeContractionError raised, get_scope_at rewinds culprit to "Mustard".
  12. Summary + Further reading: link explanation/superseded-chain-no-recovery.md, explanation/scopes-and-world-scope.md, explanation/agm-belief-revision.md; how-to/contract-a-belief.md, how-to/trace-dependency-cascade.md.

API surface — use VERBATIM (verified against src/doxastica/core.py):
  - Imports: `from uuid import uuid7` and `from doxastica import BeliefFilter, WORLD_SCOPE_ID, InMemoryBackend, MemoryCore, EdgeType, WorldScopeContractionError`.
  - Belief identity is the (scope_id, belief_id) pair; a second revise to the same pair SUPERSEDES the prior tail (no duplicate).
  - revise/expand/contract/add_edge all take `source_event_id: UUID` (caller-minted uuid7()). contract returns None; is a vacuous no-op on an absent belief; raises WorldScopeContractionError on WORLD_SCOPE_ID.
  - get_impact(belief_state_id) traverses DERIVED_FROM/DEPENDS_ON INBOUND (dependents) and returns ImpactResult(reached, frontier, truncated). Edge direction: dependent --DERIVED_FROM--> basis; call get_impact(basis.state_id) to find dependents.
  - get_scope_at(scope_id, as_of_event_id) is an INCLUSIVE cut and rewinds superseded beliefs.
  - query_scope(scope_id, BeliefFilter(), include_retracted=False) returns the current base (one tail per belief).
  - get_revision_chain(belief_id) is cross-scope by belief_id, oldest-first.

Prohibited in the published prose (per CLAUDE.md standing rule): NO GSD/process references, NO
decision IDs (e.g. D-03, BACK-01), NO phase numbers. Keep only real paper/RFC citations. Do not
claim doxastica does value-semantic consistency checking or deduction.
