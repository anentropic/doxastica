<objective>
Extend the port `traverse` primitive with a keyword-only `direction: Literal["in","out"] = "out"`
parameter and implement it on BOTH backends — the ONE genuine new capability in Phase 5 (D-05).
`direction="out"` (default) preserves the existing outgoing walk; `direction="in"` walks edges INTO
`start` (predecessors), which is what `get_impact` (Plan 03) needs to walk a belief's dependents
AGAINST the stored arrows (D-04).

Purpose: `get_impact` (EDGE-02) cannot be expressed without an incoming-direction traversal. The
default `"out"` is a CROSS-PHASE CONTRACT: it keeps the 9 existing positional `traverse` parity
calls green AND keeps the not-yet-written Phase 6 `get_scope_at` (outgoing) caller compiling
unchanged.

Output: extended `BackendPort.traverse` signature + docstring; reverse-adjacency walk in the
in-memory backend; arrow-flipped Cypher in the ladybug backend; updated `docs/backend-contract.md`
(BACK-04); reverse-direction parity tests across both backends.
</objective>
