<objective>
Implement `MemoryCore.get_impact(belief_state_id, depth=None)` (EDGE-02): the bounded, cycle-safe
contraction-cascade read that returns the set of `BeliefState`s that DEPEND ON the given node
(D-01) — its downstream dependents — by composing `traverse(..., direction="in")` (from Plan 01)
over the cascade edge set `{DEPENDS_ON, DERIVED_FROM}` (D-03), hydrating the result into full frozen
models, and packaging it as the locked `ImpactResult(reached, frontier, truncated)` (D-02).

The single most likely parity bug this plan must defeat (RESEARCH Pitfall 1): ladybug `traverse`
returns `state_id`-only rows while in-memory returns full props, so `get_impact` CANNOT hydrate
`reached` directly. This plan adopts RESEARCH Option A — re-fetch full props per reached `state_id`
via the already-parity-tested `match_nodes`, then `_hydrate` — keeping the core driver-blind and the
output byte-identical across backends.

Purpose: complete EDGE-02, the cascade MECHANISM the Phase 7 Relevance/Core-Retainment POSTULATE
tests run against. Phase 5 tests the MECHANISM only — the AGM postulate tests are Phase 7, NOT here.

Output: `MemoryCore.get_impact` body + `_CASCADE_EDGE_TYPES`; the cascade mechanism + property tests
in `tests/test_cascade.py`.
</objective>
