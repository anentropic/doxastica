    `SpineMachine`) modelling the spine against a shadow oracle. Structure:
    - `@initialize()` constructs a fresh backend + `MemoryCore(backend)` once per example and inits
      `self.model: dict[tuple[str, str], Any] = {}` (the shadow current per (scope, belief)).
    - `Bundle`s `scopes` and `beliefs` track created ids so later rules draw real ids (not random
      misses). A scope-creation rule and a belief-id-creation rule (or strategy-drawn small id pools)
      feed the bundles. Exclude WORLD_SCOPE_ID from the ordinary scope pool used by revise/expand.
    - `@rule` for `revise` (and an equivalent `expand` rule): draw a scope from `scopes`, a belief
      from `beliefs`, a json-serializable value, and a `source_event_id` from a strategy that
      DELIBERATELY produces collisions sometimes (e.g. draw from a small fixed pool of uuid7s) to
      exercise the state_id tiebreak (Pitfall 6). Call the op, then mirror it into the shadow:
      `self.model[(scope, belief)] = value`.
    - `@rule` for `contract`: `@precondition` gating it to (scope, belief) pairs that currently have a
      value in the shadow model (so vacuity vs action is exercised on both sides); call
      `core.contract(...)`, then delete the key from the shadow model (retracted ⇒ no active current).
    - A separate `@rule` (or a plain unit assertion outside the machine) asserting
      `core.contract(WORLD_SCOPE_ID, ...)` raises `WorldScopeContractionError` and leaves state
      untouched (route world-scope contraction to its own assertion, per CLAUDE.md @precondition note).
    - `@invariant` #1 (the keystone, CHAIN-03 / SC3 / D-01): for every (scope, belief) in the shadow
      model, `core._current(scope, belief)` is non-None, decodes to the shadow value, AND equals the
      HAS_REVISION chain tail for that (scope, belief) — i.e. derived-current is TOTAL and
      SINGLE-VALUED and agrees with the chain tail (compute the chain tail as the ordering-max active
      state among that belief's states filtered to the scope). For (scope, belief) NOT in the model,
      `core._current` is None (retracted/absent).
    - `@invariant` #2 (CHAIN-02 chain immutability): track a per-example monotonic counter of total
      BeliefState count (via `match_nodes("BeliefState", {})` or summing get_revision_chain lengths)
      and assert it never DECREASES across steps — no op deletes a state. (HAS_REVISION immutability
      follows: no edge-delete primitive exists; assert the chain length per belief is also
      non-decreasing.)
    - Run on BOTH backends: parametrize the machine's backend construction mirroring conftest
      (`params=["memory","ladybug"]` + `importorskip("ladybug")` so the ladybug case SKIPS, not
      fails, when the driver is absent). Expose the machine to pytest via the
      `XxxMachine.TestCase` auto-collected class (or `run_state_machine_as_test`), once per backend.
    Keep Hypothesis settings reasonable (bounded steps/examples for sub-suite runtime). This file has
