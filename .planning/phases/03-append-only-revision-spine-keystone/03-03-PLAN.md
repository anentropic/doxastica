  <action>
    Create `tests/test_revision_spine.py`. Module header mirrors `test_backend_parity.py`
    (`from __future__ import annotations`; `from typing import TYPE_CHECKING`; `import pytest`;
    `if TYPE_CHECKING: from doxastica.ports import BackendPort`). Import `MemoryCore`,
    `WorldScopeContractionError`, `WORLD_SCOPE_ID` from `doxastica`. Every test takes
    `backend: BackendPort` and constructs `core = MemoryCore(backend)` from the injected fixture port
    (NOT `MemoryCore.in_memory()` — that would ignore the parametrized ladybug backend; this is the
    one deliberate deviation from the bare-port parity tests). Use a small helper to mint
    `source_event_id` via `uuid.uuid7()`. Write these tests, each asserting the behavior on whatever
    backend the fixture supplies:
    - `test_get_or_create_scope`: creating "alice" returns Scope(is_world=False); calling twice
      returns the same scope and creates only one node (assert via a follow-up get returning the same).
    - `test_world_contract_raises`: `pytest.raises(WorldScopeContractionError)` around
      `core.contract(WORLD_SCOPE_ID, "b", uuid.uuid7())`; ALSO assert no write happened (e.g.
      `core.get_revision_chain("b")` is empty afterward) — proves the guard fires before any write.
    - `test_cross_scope_divergence` (SCOPE-03): revise the same belief_id with different values in
      two scopes; assert each scope's current (via get_revision_chain filtered by scope, or a revise
      return) reflects its own value, not the other's.
    - `test_belief_state_split` (CHAIN-01): two revises of the same belief_id produce two distinct
      BeliefStates (distinct state_id) but share the belief_id; the chain has length 2.
    - `test_revise_supersedes` (OPS-01): first revise lays no supersede (chain len 1); second revise
      supersedes the first (chain len 2, ordered).
    - `test_expand_equals_revise` (OPS-02): `core.expand(...)` and `core.revise(...)` on equivalent
      fresh inputs produce structurally identical results (same status=active, same chain growth).
    - `test_contract_vacuity_and_acts` (OPS-03): contract on a never-asserted belief returns None and
      leaves the chain empty (vacuity); contract after a revise returns None and appends exactly one
      status=retracted state whose value equals the prior current value.
    - `test_revision_chain_order` (HIST-02): several revises (use uuid7 source_event_ids, include at
      least one pair sharing a source_event_id to exercise the state_id tiebreak, Pitfall 6); assert
      get_revision_chain returns them in (source_event_id, state_id) order.
    - `test_brace_value_round_trips` (DEF-02-01): revise with `value={"x": 2}`; assert the returned
      BeliefState.value == {"x": 2} AND get_revision_chain's last state .value == {"x": 2} — the brace
      shape that ladybug otherwise corrupts.
    - `test_retracted_value_byte_identical_to_superseded` (Pitfall 2): revise then contract; assert
      the retracted state's value equals the superseded state's value (no double-encode artifacts).
    - `test_world_scope_is_world_bool` (Pitfall 4): get_or_create_scope(WORLD_SCOPE_ID) returns a
      Scope whose is_world is a real `bool` True on both backends.
    Do NOT write any production code; do NOT touch core.py. These tests are RED until 03-02 lands —
    that is expected and correct (Nyquist scaffold).
