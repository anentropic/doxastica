<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MemoryCore/caller → LadybugBackend → Cypher → lb.Connection | Caller-supplied namespace, node ids, props, and depth bound become a Cypher string + parameters; the embedded DB is in-process (no network/auth). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | namespace identifier interpolated into DDL/Cypher | mitigate | `_validate_namespace` enforces `^[A-Za-z_][A-Za-z0-9_]*$` before ANY interpolation (D-04); the one place `$param` cannot apply gets an explicit guard. Tested by test_namespace_rejects_unsafe. |
| T-02-02 | Tampering | belief-data values / node ids in Cypher | mitigate | All data flows through `$param` binds — never string-interpolated (value opacity, BACK-04 §6). Verified by the spike. |
| T-02-03 | Tampering | the var-length depth bound interpolated into `*1..N` | mitigate | The bound is an `int` from the typed port signature; assert `>= 0`; never a string. `$param` is rejected there (Pitfall 2), so the validated-int is the sanctioned form. |
| T-02-04 | Denial of Service | unbounded var-length traversal cost | mitigate | `traverse` returns the node SET via `ACYCLIC` (node-distinct), not enumerated paths — no combinatorial blow-up; `var_length_extend_max_depth` bounds engine recursion. |
| T-02-05 | Repudiation/Tampering | partial multi-write leaving inconsistent state | mitigate | `unit_of_work` wraps writes in BEGIN/COMMIT/ROLLBACK (serializable WAL); rollback verified by test_unit_of_work_rollback. |
| T-02-SC | Tampering | npm/pip/cargo installs | accept | Zero new packages this phase — `ladybug` 0.17.1 is already installed and audited OK (RESEARCH Package Legitimacy Audit); this plan only imports it, does not install it. |
</threat_model>
