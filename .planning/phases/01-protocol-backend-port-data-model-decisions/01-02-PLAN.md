## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-IV | Tampering / Info disclosure | `BeliefFilter` field set | mitigate (by design) | DATA-02: `BeliefFilter` is a CLOSED typed filter over exactly four core-owned fields — no free `str`, no arbitrary-property predicate, nothing interpolated into a query. This phase's typed surface IS the injection mitigation; the model makes a triple-structure leak unrepresentable. |
| T-01-OPAQUE | Tampering | `BeliefState.value: Any` | accept (by design) | Core treats `value` as an opaque JSON-encoded blob it never inspects, evaluates, or builds a query from. Opacity is the control — no eval, no query-construction from value content. |

Note: zero-network, zero-auth, no storage code. The load-bearing security property here is
input-validation by construction (V5 ASVS) — the closed filter and opaque value, both encoded
as frozen types. No runtime threats apply.
