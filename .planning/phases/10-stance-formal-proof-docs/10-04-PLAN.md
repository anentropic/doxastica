<objective>
Wave 2 — close the phase: export `Stance` (D-13), tell the stance story in the Cluedo
tutorial (SC5 / DOCS-01), and confirm dual-backend conformance stays green with correct
SKIP-not-fail behavior (SC4 / D-12). This runs after the three Wave-1 test tracks land so the
SC4 confirmation exercises the fully-widened suite.

D-13 is the ONE deliberate, behavior-neutral `src/` change this phase makes: `Stance` is
already a public parameter type on `revise`/`expand`, yet — unlike its sibling `Status` — it is
not importable from the package root, so the tutorial would need `from doxastica.models import
Stance`, an internal-path leak. Adding one name to `__all__` changes no behavior.

The genuine explanatory hazard (D-09) is conflating stance with scope: stance is a
*within-scope* epistemic gradient (`doubted < suspected < believed < certain`); the
certain/provisional **scope** split is a *cross-scope* distinction. The tutorial must hold
these apart and reconcile them explicitly. Ordinal comparison is a READER-side concern (D-10) —
the core stores/returns stance and never interprets it. Published docs carry no
GSD/decision-ID/phase-number/requirement-ID leakage — real paper/RFC citations only (D-11).

Purpose: ship the export + the stance narrative + the SC4 parity confirmation.
Output: `Stance` export, an extended tutorial, and a documented dual-env SKIP proof.
</objective>
